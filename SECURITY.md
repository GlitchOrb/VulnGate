# Security Policy

> @GlitchOrb

## Supported Versions

Security fixes are applied on the latest release line first.

## Reporting a Vulnerability

Preferred channels:

1. GitHub Security Advisory (private disclosure)
2. If advisory is unavailable, open a private report with:
   - affected version
   - reproduction steps
   - impact assessment
   - suggested mitigation (optional)

Please do not publish exploit details before a coordinated fix window.

## Response Targets

- Initial triage response: within 5 business days
- Status update cadence: at least weekly until resolved

## Hardening Principles

- Offline-first by default
- Minimal dependency surface
- No external SaaS requirement for MVP
- Signed artifacts/attestations when keys/identity are configured
